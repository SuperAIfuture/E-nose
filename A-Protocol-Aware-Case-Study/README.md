# Reproducibility Materials

This package provides the materials needed to reproduce the Protocol-B baseline rerun reported in the manuscript.

## Directory Layout

- `code/`
  - `parse_uci_drift.py`: parser for UCI-270 and UCI-224 batch data files.
  - `feature_routes.py`: preprocessing routes and model builders used in the Protocol-B suite.
  - `run_protocol_b_suite.py`: end-to-end runner for the shared-grid Protocol-B baseline suite.
- `data/raw/`
  - UCI-270 raw batch data used for the main benchmark analysis.
  - UCI-224 raw batch data retained for secondary reproducibility context.
- `results/reference/`
  - Reference result tables used to cross-check the reported numerical analyses.
- `results/verification/`
  - Verification result tables generated by running the packaged analysis code.

## Environment

Use Python 3.10 or later and install:

```bash
pip install -r requirements.txt
```

## Run Command

From this directory:

```bash
python code/run_protocol_b_suite.py \
  --data-root data/raw/uci270 \
  --output-dir results/verification \
  --dataset-name uci270 \
  --random-seed 42
```

Expected primary outputs:

- `results/verification/protocol_b_baseline_per_target_verification.csv`
- `results/verification/protocol_b_baseline_tuning_log_verification.csv`
- `results/verification/protocol_b_baseline_summary_verification.csv`
- `results/verification/table1_submission_ready_verification.csv`
- `results/verification/summary_vs_reference.csv`

## Analysis Policy

The runner follows the Protocol-B consistent-policy setup reported in the manuscript: target batches 2-10, a shared grid for `C` and `gamma`, latest-prior-batch validation, and the target-batch-2 row-level stratified fallback.
